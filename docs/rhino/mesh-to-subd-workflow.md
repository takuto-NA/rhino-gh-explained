# メッシュ原型 → SubDへ（QuadRemeshを軸に“作り直す”）

製造に不向きなメッシュ顔データは、頑張って修復するより **編集可能なトポロジに作り直す**方が速く、後工程に強いです。  
Rhino 8では、基本ルートは **`QuadRemesh` → SubD編集** です。

## ゴール（この章で得たい状態）

- 有機形状（顔/耳）を **SubDとして編集できる**\n- 目の周り・口・鼻翼・耳付け根など、**曲率変化の大きい場所に“流れ”がある**\n- 後でやる「厚み付け」「前後分割」「首開口」「穴あけ」「コネクタ追加」が壊れにくい

## なぜQuadRemesh→SubDが強いか

- メッシュの“悪い三角形”を整理し、**四角面中心の流れ**を作れる\n- SubDは有機形状の調整が速く、対称や局所のエッジ制御がしやすい\n- 必要なら **ToNURBS** でNURBSへ落とせる（ただし面数は増える）

## 手順の型（おすすめ）

### 1) 入力メッシュを“参照”として固定する

- 原型メッシュは残す（比較用）\n- 作業対象は複製して進める\n
### 2) 重要な“区切り”を先に考える（エッジフロー）

顔は、以下が特に重要です。

- **目の輪郭**（目の穴を後で開ける前提でも、輪郭の流れを作る）\n- **口周り**\n- **鼻翼〜頬**\n- **耳の付け根**（薄くなりやすく、割れやすい）\n- **首の付け根**（大開口を作るので、後で補強を載せる場所でもある）

この“流れ”が無いと、後段の穴あけ・分割・補強を入れた時に形が崩れやすいです。

### 3) QuadRemeshで大枠のトポロジを作る

- 分割・穴あけ・補強を見据えて、必要なら先に形状を分けてからRemeshする\n- 細部より、まずは「崩れない大枠」を優先する\n
### 4) SubDとして編集する（形状を整える）

やることは大きく2つです。

- **シルエットを合わせる**（横顔、正面、斜め）\n- **局所の“張り”を調整**（エッジ追加/クリーズなど）

目や口の“穴”はこの段階で無理に開けず、**後段で基準を持った上で開ける**方が運用が楽です。

### 5) いつToNURBSにするか（判断基準）

- **SubDのまま進める**: 造形・調整が主で、有機形状を維持したい\n- **ToNURBSへ**: 機構CADとの当たり面を作る、明確な平面/円筒/ソリッドと結合したい\n
※ ToNURBSは面が増えるので、\n+「機構との接合部や当たり面だけをNURBS/ソリッドで作る」方が破綻しにくいことが多いです。

## 次にやること
