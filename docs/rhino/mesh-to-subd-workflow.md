# メッシュ原型 → SubDへ（QuadRemeshを軸に“作り直す”）

製造に不向きなメッシュ顔データは、修復に時間をかけるより **編集可能なトポロジに作り直す**方が効率的で、後工程に適しています。  
Rhino 8では、基本ルートは **`QuadRemesh` → SubD編集** です。

> 先に概念を押さえたい場合: `subd-vs-nurbs-brep.md`（SubD / NURBS / Brep の違い）

## ゴール（この章で得たい状態）

- 有機形状（顔/耳）を **SubDとして編集できる**
- 目の周り・口・鼻翼・耳付け根など、**曲率変化の大きい場所に“流れ”がある**
- 後でやる「厚み付け」「前後分割」「首開口」「穴あけ」「コネクタ追加」が壊れにくい

## QuadRemesh→SubDが有効な理由

- メッシュの“悪い三角形”を整理し、**四角面中心の流れ**を作れる
- SubDは有機形状の調整が速く、対称や局所のエッジ制御がしやすい
- 必要なら **ToNURBS** でNURBSへ落とせる（ただし面数は増える）

## 手順の型（おすすめ）

### 1) 入力メッシュを“参照”として固定する

- 原型メッシュは残す（比較用）
- 作業対象は複製して進める

Rhinoコマンド例:

- `Copy`（作業用を作る）
- `Lock`（原型をロックして誤操作を防ぐ）
### 2) 重要な“区切り”を先に考える（エッジフロー）

顔は、以下が特に重要です。

- **目の輪郭**（目の穴を後で開ける前提でも、輪郭の流れを作る）
- **口周り**
- **鼻翼〜頬**
- **耳の付け根**（薄くなりやすく、割れやすい）
- **首の付け根**（大開口を作るので、後で補強を載せる場所でもある）

この“流れ”が無いと、後段の穴あけ・分割・補強を入れた時に形が崩れやすいです。

### 3) QuadRemeshで大枠のトポロジを作る

- 分割・穴あけ・補強を見据えて、必要なら先に形状を分けてからRemeshする
- 細部より、「崩れない大枠」を優先する

Rhinoコマンド例:

1. `QuadRemesh` を実行
2. 顔の全体が入る程度の解像度からスタート（細部は後で足す）
3. 必要なら対称を意識した向きで調整（正中を意識）

> スクショ位置案: `QuadRemesh` のオプション画面（ターゲット面数/対称など）
### 4) SubDとして編集する（形状を整える）

やることは大きく2つです。

- **シルエットを合わせる**（横顔、正面、斜め）
- **局所の“張り”を調整**（エッジ追加/クリーズなど）

Rhinoコマンド例:

- `ToSubD`（必要に応じてSubDへ変換）
- `Reflect` / `Mirror`（対称の確認・作業）
- `InsertEdge`（エッジループを追加して形を支える）
- `Crease`（シャープさを残したい場所）

目や口の「穴」はこの段階で無理に開けず、**後段で基準を持った上で開ける**方が運用が安定します。

### 5) いつToNURBSにするか（判断基準）

- **SubDのまま進める**: 造形・調整が主で、有機形状を維持したい
- **ToNURBSへ**: 機構CADとの当たり面を作る、明確な平面/円筒/ソリッドと結合したい

※ ToNURBSは面が増えるので、「機構との接合部や当たり面だけをNURBS/ソリッドで作る」方が破綻しにくいことが多いです。

## チェックリスト（SubD化の品質）

- [ ] 原型メッシュと比較して、シルエットが崩れていない（正面/側面/斜め）
- [ ] 目/口/鼻翼/耳付け根/首付け根に“流れ”がある（後で穴/分割/補強しても破綻しにくい）
- [ ] 不要な細部を追いすぎていない（大枠を優先）
- [ ] ToNURBSが必要な箇所は“当たり面”など局所だけに限定する方針がある
