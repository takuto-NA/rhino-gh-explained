# SubDを他CADに渡すとどうなる？（STEP/IGES/STL/3MFと相性）

**Q: SubDは他CADでの編集に適していますか？**

A: **SubDそのものはCADのソリッド編集と相性が悪い場合が多い**です。原因は、SubDをネイティブに扱えるCADが限られ、結局 **メッシュ扱い**になるか、Rhino側で **NURBS/Brepに変換（=面が増える）**して渡すことになるためです。

## 何が起きるか（読み込み先での扱い）

**Q: 読み込み先のCADでSubDはどう扱われますか？**

A: 以下の2つのパターンが典型的です：

**パターンA: SubDを直接読めない場合**
- **メッシュボディ**として認識されます。多くのCADのメッシュ編集は「切る/測る/参照する」用途が中心で、フィレットや精密な編集は得意ではありません。

**パターンB: Rhinoで `ToNURBS` して渡す場合**
- **Brepとしては入りますが、非常に重くなります**。SubD面の1枚1枚がNURBSパッチになるためです。その結果、ファイルが重くなり、ブーリアンやフィレットが不安定になり、後編集が困難になります。

## 各形式との相性

**Q: STEPとの相性は？**

A: **受け渡し自体は可能ですが、CADでの快適な編集は難しくなりやすい**です。STEPは本来Brep（NURBS面＋トリム）の交換に適しており、SubDから変換すると面数が膨大になりがちです。

**Q: IGESとの相性は？**

A: IGESもNURBS交換用ですが、STEPよりバラバラになりやすく、属性が落ちやすいため、可能なら **STEP優先**を推奨します。

**Q: STL / 3MF との相性は？（3Dプリント用途）**

A: **3Dプリント用途なら、最終的にメッシュにして出力するのが自然**です。
- **STL**: 高い互換性がありますが、単位やメタ情報の扱いが弱いです。
- **3MF**: 単位やメタ情報を保持しやすく、環境が揃うなら運用しやすいです。

## 解析・実務の運用

**Q: 解析に回す場合の考え方は？**

A: 目的によって変わります：
- **メッシュ解析**が前提なら、「壊れない殻（ソリッド）」であることが最優先です。
- **当たり面が厳密**な場合は、その部分だけBrepで作る価値があります。

**Q: 実務でおすすめの運用方法は？**

A: 「全部をSubD→STEPで渡して他CADで編集」しようとせず、以下のように役割分担します：
- **形（外装/有機形状）**: SubDで作る（速い）。他CADへは参照用としてメッシュで渡す。
- **CADで編集したい所（当たり面/機構）**: Rhinoで **Brep（NURBSソリッド）**として作り、STEPで渡す。

## チェックリスト（受け渡し前）

**Q: 受け渡し前に確認すべきことは？**

A: 以下を確認します：
- [ ] 相手が必要なのは「編集」か「参照」か「製造/解析」か。
- [ ] SubDをToNURBSしてSTEPにする場合、面数増加で編集性が落ちる前提を共有したか。
- [ ] 3DプリントはSTL/3MFで十分か。
