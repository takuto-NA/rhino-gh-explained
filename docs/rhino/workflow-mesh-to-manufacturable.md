# ワークフロー: 汚いメッシュ → 編集可能な曲面 → 製造/解析へ（Rhino 8）

**Q: このサイトの中心テーマは？**

A: 既存の汚いデータ（スキャン/彫刻/三角形だらけのメッシュなど）を、**編集可能な形に再構築**し、厚み・分割・穴・機構統合を行って、**3Dプリントや解析**へ回すためのワークフローを解説します。

## 全体像（おすすめの分岐）

**Q: 作業を開始する際の最初のステップは？**

A: まず、対象の **データタイプ（Mesh / Brep / SubD）を確定** してください（コマンド: `What`）。

**Q: メッシュが汚い場合の基本方針は？**

A: **修復より再構築を優先**します。`CheckMesh` や `ShowEdges` で問題を確認し、直すのが難しそうなら早めに **`QuadRemesh` → SubD** に切り替えるのが時間効率的です。

**Q: 造形と精密な加工で表現をどう使い分けますか？**

A: 役割を分担します：
- **SubD**: 造形（有機形状の作成）に適しています。
- **Brep（NURBS）**: 穴あけ/厚み付け/ソリッド演算/当たり面の作成に適しています。

**Q: 厚み付けの安全な進め方は？**

A: いきなり複雑な厚みにせず、**「均一厚みで閉じた殻」をまず作り**、その後にフランジやリブなどの補強を足すのが安全です。

**Q: 分割や穴あけの設計で注意すべきことは？**

A: **"基準（点/面）"を先に作っておく**ことです。基準があれば、形状が後から変わっても分割位置などをパラメトリックに維持できます。

**Q: 製造/解析に回す前の最終確認は？**

A: Rhino側の健全性チェックに加え、スライサーでの見え方やメッシュ品質を必ず検証します。

## 設計判断の基準

**Q: どこまでNURBSにする必要がありますか？**

A: 目的によって変わります：
- **3Dプリントが主目的の場合**: 最終出力（STL/3MF）で"壊れない殻"になっていれば、すべてをNURBSにする必要はありません。
- **解析/面品質が主目的の場合**: 正確な面構成を管理するために、NURBS/Brepに移行する価値が高まります。

このサイトでは、SubD中心の効率的なルートを主軸としつつ、必要に応じてNURBSへ移行する判断を解説します。
