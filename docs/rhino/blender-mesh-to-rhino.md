# Blenderのメッシュ（.blend）→ Rhinoへ（初心者向け：安全に受け渡す）

**Q: Blenderのファイルを直接Rhinoで開けますか？**

A: Blenderの`.blend`はRhinoで直接開けません。いったん **メッシュを書き出して（OBJなど）**、Rhinoで取り込みます。

**Q: このページの目的は？**

A: 「Blenderは分からない」前提で、**最小手順**と **詰まりどころ（単位/法線/回転/密度）**だけを整理します。

**Q: 推奨するファイル形式は？**

A: 以下の基準で選択します：
- **推奨**: OBJ（参照メッシュ用途に十分・Rhinoで扱いやすい）
- **次善**: FBX（複数オブジェクトや階層を保ちたい時）
- **非推奨（再構築用途）**: STL（単位やメタ情報が弱く、トラブルの原因になりやすい）

## 手順0：単位を揃える（mm推奨）

**Q: メッシュが小さすぎたり大きすぎたりする原因は？**

A: ほとんどの場合、単位の設定が原因です。

**Q: Blenderで単位をmmに寄せる設定は？**

A: **Scene / Units** で以下を設定します（UI名はバージョンで多少変わる場合があります）：
- Unit System: **Metric**
- Length: **Millimeters**
- Unit Scale: **0.001**（= 1 Blender unit を 1mm相当で扱う）

**Q: Rhinoで取り込み前にすべき設定は？**

A: `Units` コマンドでドキュメント単位を **mm** に固定してください。

## 手順1：Blender側で“変換ミス”を潰す

**Q: 書き出し前にBlenderで最低限やるべきことは？**

A: 以下の2点を確認します。

**1) トランスフォームを適用（スケールが1になる状態）**
- 対象メッシュを選択し、**Apply（適用）**: Rotation / Scale を実行。狙いは「書き出し後にRhinoで回転やスケールが暴れない」状態にすることです。

**2) 必要なら“軽量化”**
- GHで参照に使用する場合は、**見た目がほぼ同じでポリゴン数が少ない方が有利**です。細部は後段の曲線/面で再構築する前提で、動作する密度まで軽量化します。

## 手順2：OBJで書き出す（推奨）

**Q: OBJ書き出しのおすすめ設定は？**

A: `File > Export > Wavefront (.obj)` で以下の設定を推奨します：
- **Selection Only**: ON（必要なものだけ出す）
- **Apply Modifiers**: ON（見た目通りに出る）
- **Include Normals**: ON（陰影が破綻しにくい）
- **Scale**: 1.0

## 手順3：Rhinoで取り込み → 実寸と健全性をチェック

**Q: 取り込み後の確認事項は？**

A: 以下の3点を確認します：

**1) 取り込み**
- `Import` でOBJを読み込みます。

**2) サイズチェック（必須）**
- `BoundingBox` や `Distance` で想定寸法になっているか確認。合っていない場合はBlenderの設定やスケールを見直します。

**3) メッシュの健全性（必須）**
- `CheckMesh` でレポートを確認。
- `ShowEdges` で穴（Naked edges）を特定。
- `UnifyMeshNormals` で法線を揃える。
- 軽微な穴は `RepairMesh` / `FillMeshHoles` で修復を試みます。

## 手順4：Rhino側で“参照メッシュ”として固定する（GHの安定化）

**Q: Grasshopperで参照する際に壊れにくくする方法は？**

A: Rhino側で以下を徹底します：
- 専用レイヤ（例: `00_ref_blender_mesh`）に分ける。
- 原型は `Lock` して触らない（比較/参照専用にする）。

## 次の分岐：作り直すか / 参照から再構築するか

**Q: 取り込んだ後の進め方は？**

A: 状況に応じて以下のいずれかを選択します：

- **A) メッシュが汚い/重い場合**: `QuadRemesh` → SubD編集へ切り替えて **作り直す**。
- **B) 参照としては十分な場合**: GHで「曲線/断面」へ落として **Brep再構築** する。

## 形式の選び方（早見表）

**Q: 形式ごとの特徴は？**

A: 
- **OBJ**: 参照メッシュに最適。迷ったらこれ。
- **FBX**: 複数オブジェクト/階層を保ちたい場合に便利。
- **STL**: 3Dプリント最終出力向け。受け渡しの“中間”としては非推奨。
- **PLY**: スキャン由来などで“頂点カラー”が重要な場合のみ。
