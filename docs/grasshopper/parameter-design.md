# パラメータ設計（壊れにくい入力の作り方）

Grasshopperで“後から編集できる”状態を作るコツは、\n+**何をパラメータ（入力）として持つか**を、最初に設計することです。

## 目的別：おすすめ入力

### 形を変えたい（造形）

- **ランドマーク点**: 目頭/目尻、瞳中心、鼻先、口角、顎先など\n- **基準平面/軸**: 正中面、目の高さ面、前後基準面\n- **断面曲線**: 目の高さ断面、口の高さ断面、頬断面、正中プロファイル\n
有機形状は、制御点を直接いじるよりも\n+**断面曲線の位置・曲率・幅**をパラメータにする方が壊れにくいです。

### 厚み/クリアランスを変えたい（製造寄り）

- **厚み**（例: 1.0mm）\n- **クリアランス**（例: 0.2〜0.6mmの範囲でテストピースで決める）\n- **補強寸法**（リブ厚、フランジ幅、ボス径など）\n
## 避けたい入力（壊れやすい）

- “面の0番目”“エッジの3番目”のような **トポロジ依存の参照**\n- メッシュ原型の頂点インデックス依存\n
メッシュ→再構築のフローでは、入力形状の面構成が変わりやすいので、\n+トポロジ依存は破綻しがちです。

## パラメータの“置き場”を作る（運用）

- 定義の左側に **Inputs** グループを作る\n- スライダーは名前付きにする（Thickness_mm など）\n- 基準平面/点/曲線もまとめる\n
## 単位・公差の扱い

- GHの数値は **Rhinoドキュメント単位**で解釈される\n- 薄肉（1mm）や小さな隙間は **公差の影響**を受けやすい\n
> 関連: `../rhino/units-tolerance-and-mesh-health.md`

## チェックリスト（パラメータ設計）

- [ ] 変更したい要素が“入力（スライダー/曲線/点）”として存在する\n- [ ] 入力はトポロジ依存ではなく、基準・ランドマーク・断面曲線に寄せている\n- [ ] 厚み/クリアランスはテストピースで決める前提がある\n
