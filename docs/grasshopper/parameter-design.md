# パラメータ設計（壊れにくい入力の作り方）

## 基本方針

**Q: パラメータ設計で重要なことは？**

A: Grasshopperで「後から編集できる」状態を作るには、**何をパラメータ（入力）として持つか**を最初に設計することが重要です。

## 目的別の推奨入力

**Q: 形を変えたい（造形）場合の推奨入力は？**

A: 以下を推奨します：
- **ランドマーク点**: 目頭/目尻、瞳中心、鼻先、口角、顎先など
- **基準平面/軸**: 正中面、目の高さ面、前後基準面
- **断面曲線**: 目の高さ断面、口の高さ断面、頬断面、正中プロファイル

有機形状は、制御点を直接いじるよりも**断面曲線の位置・曲率・幅**をパラメータにする方が壊れにくいです。

**Q: 厚み/クリアランスを変えたい（製造寄り）場合の推奨入力は？**

A: 以下を推奨します：
- **厚み**（例: 1.0mm）
- **クリアランス**（例: 0.2〜0.6mmの範囲でテストピースで決める）
- **補強寸法**（リブ厚、フランジ幅、ボス径など）

## 避けたい入力

**Q: 避けるべき入力は？**

A: 以下は壊れやすいため避けます：
- "面の0番目""エッジの3番目"のような**トポロジ依存の参照**
- メッシュ原型の頂点インデックス依存

メッシュ→再構築のフローでは、入力形状の面構成が変わりやすいので、トポロジ依存は破綻しがちです。

## パラメータの管理

**Q: パラメータを管理する方法は？**

A: 以下を推奨します：
- 定義の左側に**Inputs**グループを作る
- スライダーは名前付きにする（Thickness_mm など）
- 基準平面/点/曲線もまとめる

## 単位・公差の扱い

**Q: 単位と公差の扱いは？**

A: 以下に注意します：
- GHの数値は**Rhinoドキュメント単位**で解釈される
- 薄肉（1mm）や小さな隙間は**公差の影響**を受けやすい

> 関連: `../rhino/units-tolerance-and-mesh-health.md`

## チェックリスト

**Q: パラメータ設計の確認ポイントは？**

A: 以下を確認します：
- [ ] 変更したい要素が"入力（スライダー/曲線/点）"として存在する
- [ ] 入力はトポロジ依存ではなく、基準・ランドマーク・断面曲線に寄せている
- [ ] 厚み/クリアランスはテストピースで決める前提がある
