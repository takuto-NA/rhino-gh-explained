# 実例: メッシュ原型→編集可能な外装（最小GH構成）

ここでは「汚いメッシュを参照しながら、後から編集できる外装」を作るための **最小構成の考え方**を示します。

コンポーネント名はGrasshopper標準を前提に、代表例も併記します（多少表記が違っても、カテゴリで辿れます）。

## ゴール

**Q: このワークフローのゴールは？**

A: 以下の状態を目指します：
- 入力（ランドマーク点・断面曲線・厚み・クリアランス）を変えると形が更新される
- "穴位置は後で決める""分割は後で変える"の運用に耐える

## 入力（Inputs）

**Q: 必要な入力は？**

A: 以下を推奨します：
- **基準**: 正中面、目の高さ面、前後基準面
- **ランドマーク**: 鼻先/瞳中心/口角など（Point）
- **断面曲線**: 正中プロファイル、数本の横断面（Curve）
- **数値**: Thickness_mm / Clearance_mm

## 形状生成（Shape）

**Q: 推奨される基本構成は？**

A: 以下の構成を推奨します：

1. 断面曲線群 → ロフト/ネットワーク系で外形サーフェスを作る
2. 外形をスムーズに整える（必要なら曲線側で調整）

メッシュ原型は "参照" に留めて、外形は **断面曲線で再構築**するのが壊れにくいです。

## 厚み（Shell）

**Q: 厚みの作り方は？**

A: 厚みは「均一殻 → 後で補強」が安全です。GH上では、厚み値を入力として保持します。

> 関連: `../rhino/variable-thickness-shell.md`

**Q: GHでオフセット（厚み）を作る際の標準コンポーネントは？**

A: 以下のコンポーネントを使用します：

- **曲線のオフセット**: `Offset Curve`（Curve → Util）
- **サーフェス/Brepのオフセット**: `Offset Surface`（Surface → Util）
  - 入力: Surface/Brep, Distance（mm）
  - 可能なら `Solid` / `BothSides` / `Extend` などのオプションを明示的に管理する
- **メッシュのオフセット**: `Offset Mesh`（Mesh → Util）

**Q: 運用上の注意は？**

A: 以下に注意します：
- 強曲率部は自己交差で失敗しやすいので、厚みは「均一殻→後で補強」の順が安全
- "外形を守りたい/内側を守りたい"でオフセット方向（符号）を決める

## 分割/穴/コネクタ（後段）

**Q: 分割/穴/コネクタの扱いは？**

A: 以下を推奨します：
- 分割線/基準線を入力にして、後段で更新できるようにする
- 穴は基準点＋寸法（半径/楕円）で後から確定
- コネクタは当たり面を作ってから（外装に直接貼らない）

> 関連: `../rhino/splitting-and-openings.md` / `../rhino/connectors-and-cad-integration.md`

## ベイク運用

**Q: ベイク運用の判断基準は？**

A: 以下を推奨します：
- 造形（Shape）まではGHで管理
- 製造向けの最終調整（当たり面の微修正など）はRhinoで行う場合、"どこでBakeするか" を決める

> 関連: `history-and-editing.md`
