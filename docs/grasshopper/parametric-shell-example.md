# 実例: メッシュ原型→編集可能な外装（最小GH構成）

ここでは「汚いメッシュを参照しながら、後から編集できる外装」を作るための **最小構成の考え方**を示します。

コンポーネント名はGrasshopper標準を前提に、代表例も併記します（多少表記が違っても、カテゴリで辿れます）。

## ゴール

- 入力（ランドマーク点・断面曲線・厚み・クリアランス）を変えると形が更新される\n- “穴位置は後で決める”“分割は後で変える”の運用に耐える\n
## 入力（Inputs）

- **基準**: 正中面、目の高さ面、前後基準面\n- **ランドマーク**: 鼻先/瞳中心/口角など（Point）\n- **断面曲線**: 正中プロファイル、数本の横断面（Curve）\n- **数値**: Thickness_mm / Clearance_mm\n
## 形状生成（Shape）

推奨される基本構成:

1. 断面曲線群 → ロフト/ネットワーク系で外形サーフェスを作る\n2. 外形をスムーズに整える（必要なら曲線側で調整）\n
メッシュ原型は “参照” に留めて、\n+外形は **断面曲線で再構築**するのが壊れにくいです。

## 厚み（Shell）

厚みは「均一殻 → 後で補強」が安全です。\n+GH上では、厚み値を入力として保持します。

> 関連: `../rhino/variable-thickness-shell.md`

### GHでオフセット（厚み）を作る際の標準コンポーネント

- **曲線のオフセット**: `Offset Curve`（Curve → Util）
- **サーフェス/Brepのオフセット**: `Offset Surface`（Surface → Util）
  - 入力: Surface/Brep, Distance（mm）
  - 可能なら `Solid` / `BothSides` / `Extend` などのオプションを明示的に管理する
- **メッシュのオフセット**: `Offset Mesh`（Mesh → Util）

運用上の注意:

- 強曲率部は自己交差で失敗しやすいので、厚みは「均一殻→後で補強」の順が安全
- “外形を守りたい/内側を守りたい”でオフセット方向（符号）を決める

## 分割/穴/コネクタ（後段）

- 分割線/基準線を入力にして、後段で更新できるようにする\n- 穴は基準点＋寸法（半径/楕円）で後から確定\n- コネクタは当たり面を作ってから（外装に直接貼らない）\n
> 関連: `../rhino/splitting-and-openings.md` / `../rhino/connectors-and-cad-integration.md`

## ベイク運用（実務上の判断基準）

- 造形（Shape）まではGHで管理\n- 製造向けの最終調整（当たり面の微修正など）はRhinoで行う場合、\n+  “どこでBakeするか” を決める\n
> 関連: `history-and-editing.md`
